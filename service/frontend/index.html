How it Works

This service processes historical document text to separate high-quality content from OCR noise.

1. Layout Analysis (ALTO Only)

If you upload an XML file, we use LayoutLMv3 to reorder words based on their 2D bounding boxes, ensuring multi-column text is read in the correct order.

2. Split Word Reconstruction

We automatically merge hyphenated words at the end of lines (e.g., divi- + ded → divided) using extracted hyphenation flags.

3. Quality Classification

Every line is analyzed using two models:

  • FastText: Identifies the language (e.g., English, German, Czech).
  • DistilGPT2: Calculates Perplexity (how "surprising" or garbled the text looks).

Based on these metrics, lines are categorized as:

  • Clear Fluent text in a target language.
  • Noisy Text with OCR errors or mixed fragments.
  • Trash Headers, page numbers, or severe OCR garbage.
