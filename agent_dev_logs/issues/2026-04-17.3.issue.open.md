#### 4.1 Classify Lines (GPU Bound) 🚀

This script reads the extracted text files, batches lines together 📦, and runs the FastText [^2]
and DistilGPT2 [^6] models. It uses a **CPU/GPU split architecture**:

- A single dedicated **GPU worker** holds the only DistilGPT2 instance and processes perplexity batches to prevent VRAM OOM errors.
- Multiple **CPU workers** (up to `WORKERS_MAX`, default 32) read files, run FastText and structural detectors, and submit text batches to the GPU worker via a shared queue. CPU workers poll the result dictionary while the GPU processes, running language identification concurrently.

`<doc_name>.csv`: Detailed classification results for every single line within a document, with columns:

* `file` — document identifier 🆔
* `page_num` — page number 📄
* `line_num` — line number, starts from 1 for each page 🔢
* `text` — original text of the line 📝
* `split_ws` — hyphenated word prefix at the end of the line (split word start)
* `split_we` — hyphenated word suffix at the start of the line (split word end)
* `lang` — predicted ISO language code from the FastText model ([full list](https://github.com/facebookresearch/flores/tree/main/flores200#languages-in-flores-200)) 🌐
* `lang_score` — FastText confidence score for the predicted language 🎯
* `perplex` — DistilGPT2 perplexity score of the line 📉
* `word_count` — number of whitespace-delimited tokens in the line
* `char_count` — total character count of the line
* `garbage_density` — ratio of non-alphanumeric, non-standard-punctuation characters to total line length
* `symbol` — count of words containing disallowed internal symbols (see detectors below)
* `upper` — count of words with unexpected mid-word uppercase letters
* `repeated` — count of words where a non-standard character makes up ≥ 40% of the word
* `ldl_fuses` — count of words with a letter–digit–letter sandwich (e.g., `w0rd`)
* `gibberish` — count of words flagged as gibberish (all-caps, no vowels, or extreme vowel ratio)
* `word_weird` — mean per-word weirdness score in [0, 1]; combines strange-symbol, repeated-symbol, LDL-fusion, and mid-uppercase signals weighted per token (0 = fully clean)
* `quality_score` — composite quality score in [0, 1] based on valid-word ratio, symbol ratio, perplexity, and text length; higher = cleaner
* `categ` — assigned category: **Clear** ✅, **Noisy** ⚠️, **Trash** 🗑️, **Non-text** 🔣, or **Empty** 🫙

##### CPU Pre-filter

Before any GPU or model inference, `pre_filter_line()` applies a fast CPU-side check and assigns `Empty` or `Non-text` directly, bypassing the ML pipeline entirely:

* Line is blank → **Empty**
* Fewer than 4 characters, or fewer than 3 unique non-whitespace symbols → **Non-text**
* Letter ratio below 30% of total characters → **Non-text**
* Matches the all-digits/symbols regex pattern → **Non-text**
* Otherwise → forwarded for ML classification as **Process**

##### Structural Detectors

Lines that pass the pre-filter are analysed by four structural detectors defined in `text_util_langID.py`:

| Detector | What it counts |
|---|---|
| `detect_strange_symbols` | Words containing any character that is not alphanumeric and not in the allowed set `{ . - , + ( ) " ' / _ — – : % }`. Edge punctuation is stripped before inspection. |
| `detect_letter_digit_letter` | Words with a **letter–digit–letter sandwich** — the fingerprint of OCR digit insertions mid-word (e.g., `vyt1ačená`, `nalez2í`). Legitimate patterns like `90,9g`, `80-90cm`, `26.IX.1957` do not trigger. |
| `detect_mid_uppercase` | Words with unexpected uppercase mid-word (`dalSÍ`, `obkLADem`) or an uppercase run at the start followed by lowercase (`XXWžkumu`). All-caps words and titles (`PhDr`, `MUDr`) are excluded. |
| `detect_repeated_chars` | Words where a single non-standard character makes up ≥ 40% of the word (e.g., OCR stutter like `bxxxoxx`). |
| `detect_gibberish_words` | Words of length ≥ 7 that are all-uppercase, contain no vowels, or have a vowel ratio below 15% or above 80%. |

##### Categorisation Logic (Cumulative Penalty System)

`categorize_line()` in `text_util_langID.py` uses a cumulative floating-point penalty score rather than a fixed decision tree. The full logic is evaluated as follows:

**Immediate Trash overrides** (checked first, before penalty accumulation):

* Garbage density > 0.35, or garbage density > 0.20 on lines of ≤ 3 words → **Trash**
* Perplexity > 500 **and** a structural weirdness ratio (`word_weird`) > 0.4 simultaneously → **Trash** *(hard override to catch severe, high-confidence garbage)*

**Penalty accumulation** (for lines that pass the overrides):

| Signal | Penalty added |
|---|---|
| Each word with a strange symbol (`sym_count`) | `sym_count × 0.4` |
| Two or more strange-symbol words | additional `+0.5` |
| Each LDL-fused token | `× 0.3` |
| Each mid-word uppercase word | `× 0.2` |
| Each word with repeated non-standard char | `× 0.4` |
| Each gibberish word | `× 0.5` |

**Perplexity penalty** (skipped for short phrases that are structurally clean):

A line with fewer than 5 words, whose language is in the `EXPECTED_LANGS` allowlist and whose structural penalty is zero, is treated as a "forgiven short phrase" and perplexity thresholds are not applied. For all other lines:

* Perplexity > `PERPLEXITY_THRESHOLD_MIN` (default 1500, scaled to `× 1.5` for lines < 5 words) → `+0.5`
* Perplexity > `PERPLEXITY_THRESHOLD_MAX` (default 5000) → additional `+1.0`

> [!NOTE]
> Perplexity is intentionally **not used** as a Trash signal in isolation. `distilgpt2` is an English model and
> assigns very high perplexity to legitimate short Czech strings (place names, postal codes, form-field labels),
> making it unreliable as a Trash indicator. It is applied only as an additive penalty with structural context.

**Language confidence penalty:**

* Predicted language is **not** in `EXPECTED_LANGS` and confidence < 0.60 → `+0.8`
* Predicted language **is** in `EXPECTED_LANGS` but confidence < 0.30 → `+0.5`

**Final classification** via normalized penalty:

```
normalized_penalty = total_penalties / max(1.0, word_count / 5.0)

normalized_penalty ≥ 1.2  →  Trash
normalized_penalty ≥ 0.3  →  Noisy
otherwise                 →  Clear
```

##### Quality Score Computation

The `quality_score` is a continuous confidence metric ranging from `0.0` to `1.0` (where higher is cleaner). It is calculated in `text_util_langID.py` using a linear-weighted combination of four normalized text features:

* **Valid Word Ratio (40% weight):** The proportion of structurally valid words in the line (length ≥ 3, ≥ 70% alphabetic, no strange symbols).
* **Inverse Symbol Ratio (30% weight):** Rewards clean text by penalizing non-alphanumeric character density. Computed as `1.0 - min(symbol_ratio, 1.0)`.
* **Inverse Perplexity (20% weight):** Rewards text predictability based on the DistilGPT2 output. Computed as `1.0 - min(perplexity / 5000.0, 1.0)`.
* **Normalized Text Length (10% weight):** Slightly rewards longer text lines up to a 100-character ceiling. Computed as `min(text_length / 100, 1.0)`.

**Current Effect on Categorization (v0.13.0)**
Currently, the `quality_score` **does not** affect the category assignment. As seen in `langID_classify.py`, the score is calculated strictly for CSV metadata logging and downstream analytics. The actual line categorization (`Clear`, `Noisy`, `Trash`) is entirely driven by the **Cumulative Penalty System** (which uses the `word_weird` ratio, perplexity thresholds, and individual structural flags).

##### Post-Processing Smoothing

After all lines in a document are classified and written to CSV, a final data-smoothing pass is applied before the file is finalized to prevent unnatural categorization anomalies:

1. **Header/Footer Deduplication** — Resolves edge-case flip-flopping. If the exact same text string appears multiple times across a document, all instances are harmonized to share the statistical mode (most frequent) category assigned to that string.
2. **Context Smoothing (Rolling Window)** — Applies a 3-line rolling window. If a **Noisy** line is sandwiched between two consecutive **Trash** lines, it is automatically downgraded to **Trash** to prevent isolated "noisy" categorizations in otherwise heavily corrupted regions.

#### 4.2 Aggregate Statistics (Memory Bound) 🧠

This script processes the `DOC_LINE_LANG_CLASS/` directory with CSV files in chunks 🧩 to produce
final page-level statistics. It is **CPU-bound** and parallelized with `ProcessPoolExecutor`.

For each page, the aggregation computes:

**Category counts** (from all lines regardless of category):

* `Clear`, `Noisy`, `Trash`, `Non-text`, `Empty` — integer count of lines in each category

**Totals** (summed over lines classified as Clear, Noisy, or Trash only — Empty and Non-text excluded):

* `total_word_count` — total number of words across scoreable lines
* `total_char_count` — total number of characters across scoreable lines

**Averages** (mean over the same Clear/Noisy/Trash lines):

* `avg_garbage_density` — mean garbage density ratio
* `avg_lang_score` — mean FastText confidence score
* `avg_perplex` — mean DistilGPT2 perplexity score
* `avg_symbol` — mean strange-symbol word count
* `avg_upper` — mean mid-uppercase word count
* `avg_repeated` — mean repeated-char word count
* `avg_ldl_fuses` — mean LDL-fusion word count
* `avg_gibberish` — mean gibberish word count
* `avg_word_weird` — mean per-word weirdness ratio in [0, 1]; 0 = fully clean, lower is better 📉
* `avg_quality_score` — mean composite quality score in [0, 1]; higher = cleaner OCR output 📈

**Language profile:**

* `main_lang` — the statistical mode (most frequent) language predicted for the page, excluding lines where FastText returned `N/A` or `unknown`

> [!NOTE]
> `avg_*` columns and `main_lang` will be `NaN` / `unknown` for pages whose only lines are
> Empty or Non-text (i.e., pages with no scoreable text content).

All numeric averages are rounded to 4 decimal places; totals are stored as integers.

**1. CPU Pre-filtering**
Before reaching any machine learning models, a fast CPU check assigns lines directly to `Empty` (if completely blank) or `Non-text`. A line is flagged as `Non-text` if it has fewer than 4 characters, lacks sufficient alphabetical letters (< 30%), or is predominantly composed of digits and symbols.

**2. Language Remapping**
To prevent false penalties on short strings, FastText language predictions that fall outside the `EXPECTED_LANGS` or `TRUSTED_FOREIGN_LANGS` lists are automatically remapped to the primary expected language (e.g., `ces`) Their confidence score is also bumped to a minimum `LANG_SCORE_CLEAR` (0.75)

**3. Structural Detection & Quality Scoring**
Lines passing the pre-filter are analyzed by structural detectors that count anomalies like strange symbols, letter-digit-letter fusions, mid-word uppercase letters, and gibberish (e.g., missing vowels). These metrics compute a composite `quality_score` from 0.0 to 1.0 using a weighted formula:
*   Valid word ratio: 30% weight
*   Inverse symbol ratio: 20% weight
*   Inverse word "weirdness": 20% weight
*   Inverse perplexity (DistilGPT2): 20% weight
*   Normalized text length: 10% weight

**4. Category Assignment**
The actual category is assigned using a mix of hard overrides and the computed `quality_score`:
*   **Immediate Trash Overrides:** A line is instantly labeled `Trash` if it suffers from severe OCR corruption, such as a garbage density > 0.35, high fragmentation (words < 2.5 characters with high weirdness), extreme vowel ratios (< 10% or > 90%), or high weirdness combined with high perplexity.
*   **Thresholds:** If no overrides are triggered, the `quality_score` dictates the label: scores `< 0.40` become `Trash`, scores `< 0.70` become `Noisy`, and anything `≥ 0.70` is marked `Clear`.

**5. Post-Processing Smoothing**
Finally, the script applies contextual smoothing to the entire document. Identical text strings repeated across a document are harmonized to share their most frequent (mode) category, fixing header/footer flip-flopping. Additionally, a rolling window downgrades any `Noisy` line sandwiched between two `Trash` lines into `Trash`.

### 1. Configurable perplexity model (`langID_classify.py` + `config_langID.txt`)

The GPU inference worker, previously wired to `distilgpt2` implicitly, now accepts a `model_name` argument and is driven by a new config key:

```ini
MODEL_NAME = Qwen/Qwen2.5-0.5B
```

`gpu_inference_worker` receives this as a parameter and calls `AutoModelForCausalLM.from_pretrained(model_name)`, so any HuggingFace causal LM can be swapped in without touching source code. The main function reads `MODEL_NAME` with a fallback to `Qwen/Qwen2.5-0.5B` if the key is absent.

The practical implication: Qwen2.5-0.5B scores clean text at considerably lower perplexity than distilgpt2 did, so the hardcoded thresholds that worked for distilgpt2 would over-trash Czech lines when used with Qwen. This motivates the next change.

---

### 2. Four new configurable thresholds in `[TEXT_UTILS]`

Previously hardcoded inside `categorize_line`, these four decision boundaries are now read from the config (with their old values as fallbacks where they existed):

| Key | New default | Previous hardcoded value | Role |
|---|---|---|---|
| `PERPLEXITY_THRESHOLD_MAX` | `1000.0` | `3000.0` | Max perplexity for normalization of score |
| `CATEG_PPL_SHORT_MAX` | `700.0` |`2000.0` | Max perplexity for short lines before Trash/Noisy split |
| `CATEG_PPL_WEIRD_MAX` | `400.0` | `1000.0` | Perplexity ceiling when `weird_ratio > 0.15 |

The lower PPL thresholds (`700` / `400`) reflect Qwen's tighter perplexity range on clean Czech text and are the minimum needed to keep the Trash/Noisy/Clear split proportions stable after the model switch.

---

### 3. Single-character word processing adapted to Czech (`text_util_langID.py`)

This is the most structurally significant change and touches two separate functions.

**`score_word()` — expanded single-character whitelist**

The original function had no special handling for length-1 tokens. Czech text is unusually rich in single-letter prepositions (`v`, `z`, `k`, `s`) and archival abbreviation initials (`p.` = *pan*, `d.` = *doktor/den*, etc.). Without a whitelist these would have driven `weird_ratio` up on otherwise clean lines.

The updated logic for `len(core) == 1`:

```python
if core in "aAiIoOuUvVzZkKsSpPbBjJdDrRnNmMtT" or '.' in word:
    return 0.0    # whitelisted grammatical / archival single letter
if core.isdigit():
    return 0.25   # tolerable penalty for isolated measurements
if not core.isalpha():
    return 0.0    # surviving punctuation separators (e.g. '-')
return 0.85       # severe weirdness for any other stray letter
```

The high penalty (`0.85`) for unlisted isolated letters means that genuinely fragmented garbage like `C A s 8.` still accumulates enough `weird_ratio` to be caught downstream.

**`categorize_line()` — new "Catch 7" fragmentation guard**

A new check was added after the short-line perplexity catch:

```python
single_char_ratio = sum(1 for w in text_source.split()
                        if len(w.strip(_STRIP_CHARS)) <= 1) / wc
if wc >= 3 and single_char_ratio >= 0.50 and weird_ratio > 0.15:
    return "Trash"
```

The `weird_ratio > 0.15` guard is the key calibration: a line like `v z k Praha` has `single_char_ratio = 0.75` but its whitelisted tokens produce a near-zero `weird_ratio`, so it is not Trash-classified. A line like `C A s 8.` has both a high ratio and a high `weird_ratio` and is correctly trashed.

A related relaxation was applied to the average-word-length fragmentation check to stop valid measurement lines (e.g. `145 mm, pr...`) from being caught:

```python
if wc >= 5 and avg_word_len < 2.0 and weird_ratio > 0.1:   # added weird_ratio guard
    return "Trash"
```

**`pre_filter_line()` — numeric-heavy lines preserved**

An early-exit was added so that lines whose digit share exceeds 40% (dates, dimension strings, catalogue numbers) are forwarded for ML scoring rather than discarded as Non-text:

```python
if sum(c.isdigit() for c in clean_text) / n_chars > 0.4:
    return "Process", clean_text
```

def categorize_line(
        quality_score: float,
        text_source: str,
        wc: int,
        weird_ratio: float,
        vowel_ratio: float,
        perplexity: float,
        original_lang_score: float | None = None
) -> str:
    if wc == 0 or not text_source.strip():
        return "Empty"

    g_density = compute_garbage_density(text_source)
    if g_density > CATEG_GARBAGE_DENSITY_HIGH or (
            wc <= CATEG_GARBAGE_SHORT_WC and g_density > CATEG_GARBAGE_DENSITY_SHORT):
        return "Trash"

    avg_word_len = sum(len(w.strip(_STRIP_CHARS)) for w in text_source.split()) / wc if wc > 0 else 0
    if wc >= 5 and avg_word_len < 2.0 and weird_ratio > 0.1:
        return "Trash"

    # Catch 1: High perplexity on short lines
    if perplexity > CATEG_PPL_SHORT_MAX and wc < 5:
        if not all(c in "IVXLCDMivxlcdm.-, " for c in text_source):
            if g_density < 0.1 and weird_ratio < 0.20:
                return "Noisy"
            return "Trash"

    # We pre-calculate single_char_ratio for Catch 7 and Fix D.
    single_char_ratio = (
        sum(1 for w in text_source.split() if len(w.strip(_STRIP_CHARS)) <= 1) / wc
        if wc > 0 else 0
    )

    # Catch 7 / FIX B: Single-character fragmentation (spaced out text)
    if wc >= 3 and single_char_ratio >= 0.50 and weird_ratio > 0.15:
        normal_word_count = sum(
            1 for w in text_source.split()
            if len(w.strip(_STRIP_CHARS)) >= 4
        )
        if normal_word_count >= 3:
            return "Noisy"
        return "Trash"

    # Catch 2: Extremely skewed vowel ratios
    if len(text_source) > 5 and (vowel_ratio < 0.1 or vowel_ratio > 0.9):
        return "Trash"

    # Catch 3: High overall word weirdness
    if weird_ratio >= 0.25:
        return "Trash"

    # Catch 4 / FIX C: Moderately high weirdness combined with high perplexity
    if weird_ratio > 0.15 and perplexity > CATEG_PPL_WEIRD_MAX:
        valid_wr = compute_valid_ratio(text_source)
        if valid_wr >= 0.5:
            return "Noisy"
        return "Trash"

    if quality_score < CATEG_TRASH_SCORE_MAX:
        return "Trash"
    if quality_score < CATEG_NOISY_SCORE_MAX:
        return "Noisy"

    # FIX D: lines with a noticeable spaced-letter OCR pattern
    if single_char_ratio >= 0.25 and wc >= 3:
        return "Noisy"

    # FIX E-2: Pre-remap confidence gate
    if original_lang_score is not None and original_lang_score < LANG_SCORE_ROUGH:
        return "Noisy"

    return "Clear"
```

def compute_quality_score(
        valid_word_ratio: float,
        symbol_ratio: float,
        perplexity: float,
        text_length: int,
        weird_ratio: float,
        vowel_ratio: float = 0.40,
        garbage_density: float = 0.0,
        lang_score: float | None = None,
        gibberish_ratio: float = 0.0,
        fused_ratio: float = 0.0,
        ppl_max: float = PERPLEXITY_THRESHOLD_MAX,
        length_max: float = QS_LENGTH_MAX,
        rot_ratio: float = 0.0,
) -> float:
    """
    Compute a single quality score in [0, 1] that encodes every meaningful
    signal available for an OCR text line.
    """
    # 1. Dynamically calculate the maximum possible weight sum to prevent inflation
    total_weight = (
            QS_WEIGHT_VALID_WORD
            + QS_WEIGHT_SYMBOL
            + QS_WEIGHT_WEIRD
            + QS_WEIGHT_PERPLEXITY
            + QS_WEIGHT_LENGTH
            + QS_WEIGHT_GARBAGE
            + QS_WEIGHT_VOWEL
            + QS_WEIGHT_LANG
            + QS_WEIGHT_GIBBERISH
            + QS_WEIGHT_FUSED
    )

    norm_symbol = 1.0 - min(symbol_ratio, 1.0)
    norm_ppl = 1.0 - min(perplexity / ppl_max, 1.0)
    norm_len = min(text_length / length_max, 1.0)
    norm_weird = 1.0 - min(weird_ratio, 1.0)

    # Dynamic Garbage Penalty Guard
    active_garbage_weight = QS_WEIGHT_GARBAGE
    if text_length <= 12 and weird_ratio == 0.0:
        active_garbage_weight = active_garbage_weight / 2.0

    norm_garbage = 1.0 - min(garbage_density / max(CATEG_GARBAGE_DENSITY_HIGH, 1e-9), 1.0)

    # Vowel quality
    vr = vowel_ratio
    if vr < 0.20:
        norm_vowel = vr / 0.20
    elif vr > 0.75:
        norm_vowel = max(0.0, 1.0 - (vr - 0.75) / 0.25)
    else:
        norm_vowel = 1.0

    norm_lang = lang_score if lang_score is not None else 0.5
    norm_gibb = 1.0 - min(gibberish_ratio, 1.0)
    norm_fused = 1.0 - min(fused_ratio, 1.0)

    base_score = (
            QS_WEIGHT_VALID_WORD * valid_word_ratio
            + QS_WEIGHT_SYMBOL * norm_symbol
            + QS_WEIGHT_WEIRD * norm_weird
            + QS_WEIGHT_PERPLEXITY * norm_ppl
            + QS_WEIGHT_LENGTH * norm_len
            + active_garbage_weight * norm_garbage
            + QS_WEIGHT_VOWEL * norm_vowel
            + QS_WEIGHT_LANG * norm_lang
            + QS_WEIGHT_GIBBERISH * norm_gibb
            + QS_WEIGHT_FUSED * norm_fused
    )

    # Re-normalize if garbage weight was reduced so maximum possible score remains intact
    if active_garbage_weight != QS_WEIGHT_GARBAGE:
        base_score += (QS_WEIGHT_GARBAGE - active_garbage_weight)

    # 2. OVERRIDE INFLATION: Divide by the actual total_weight
    # This guarantees the score strictly bounds to 1.0 regardless of config experimentation.
    base_score = base_score / total_weight

    # 3. Conditional Rotation Penalty
    rot_penalty = 0.0
    if rot_ratio >= ROT_RATIO_INVERTED_MIN:
        if weird_ratio >= WEIRD_RATIO_INVERTED_MIN:
            rot_penalty = (rot_ratio * weird_ratio) * 2.0
        elif perplexity >= PPL_INVERTED_MIN:
            rot_penalty = 0.40

        # 4. SAFETY GATE: Protect highly confident readable Czech texts from false trashing
        if lang_score is not None and lang_score >= 0.90:
            rot_penalty *= 0.5

    final_score = max(0.0, base_score - rot_penalty)
    return min(1.0, final_score)
```

def categorize_line(
        qs: float,
        txt: str,
        wc: int,
        vowel_ratio: float,
        perplexity: float,
        rot_ratio: float = 0.0,
        weird_ratio: float = 0.0,
) -> tuple[str, float]:
    def _determine_category(quality_score: float, text_source: str, word_count: int,
                            vr: float, ppl: float) -> str:
        # Override 1: empty line
        if word_count == 0 or not text_source.strip():
            return "Empty"

        # Override 2: all-caps with negligible vowels is definitively unreadable
        if is_all_caps_line(text_source) and vr < 0.10:
            return "Trash"

        # Override 3: language model is highly confident — trust it over heuristics
        if ppl < 50.0 and word_count >= 3:
            return "Clear"

        # Pure score-based routing
        if quality_score < CATEG_TRASH_SCORE_MAX:
            return "Trash"
        if quality_score < CATEG_NOISY_SCORE_MAX:
            # Override 5: Near-boundary readable Czech prose promotion.
            if (quality_score >= CLEAN_PROSE_MIN_SCORE
                    and word_count >= CLEAN_PROSE_WC_MIN
                    and weird_ratio < CLEAN_PROSE_WEIRD_MAX
                    and ppl < CLEAN_PROSE_PPL_MAX):
                return "Clear"
            return "Noisy"
        return "Clear"

    categ = _determine_category(qs, txt, wc, vowel_ratio, perplexity)

    # Enforce interconnected boundaries without modifying the logical routing
    if categ == "Trash":
        aligned_score = min(qs, CATEG_TRASH_SCORE_MAX - 0.0001)
    elif categ == "Noisy":
        aligned_score = max(qs, CATEG_TRASH_SCORE_MAX)
        aligned_score = min(aligned_score, CATEG_NOISY_SCORE_MAX - 0.0001)
    elif categ == "Clear":
        aligned_score = max(qs, CATEG_NOISY_SCORE_MAX)
    else:
        aligned_score = qs

    return categ, aligned_score

### 1. The Failing Test: Why it fails & Algorithmic Options

**The Error:**
The string `"pbqdnuwmoxszeyv!!"` (or `"pbqdnuwmoxszeyv"`) is meant to test inverted/mirror scan detection and should be classified as `Trash` or `Non-text`. Instead, it is being classified as `Noisy`.

**The Root Cause:**
In `compute_quality_score`, the rotation penalty (`rot_penalty`) is highly guarded. It **only** applies if `weird_ratio >= WEIRD_RATIO_INVERTED_MIN` OR (`perplexity >= PPL_INVERTED_MIN` **and** `weird_ratio > 0.0`).
Because the string `"pbqdnuwmoxszeyv"` consists entirely of valid lowercase alphabetical characters with no unusual repeated runs and no mid-word uppercase letters, `score_word()` assigns it a weirdness score of `0.0`. Since `weird_ratio == 0.0`, the code bypasses the rotation penalty entirely, leaving the Quality Score too high.

**Algorithmic Options to Fix This:**

1. **Apply the Missing "w" Penalty (Best Option):** Your requirements state: *"Apply a weirdness penalty for words containing the letter 'w'"*. If you implement this inside `score_word()`, the string `"pbqdnuwmoxszeyv"` will naturally gain a `weird_ratio > 0.0` (because it contains a 'w'), unlocking the rotation penalty and fixing the test naturally without changing the `compute_quality_score` math.
2. **Decouple Rotation from Weirdness:** Change the rule in `compute_quality_score` so that extreme rotation ratios apply a penalty regardless of weirdness.
*Example:* `elif perplexity >= PPL_INVERTED_MIN: rot_penalty = rot_ratio * 0.5` (dropping the `and weird_ratio > 0.0` requirement).
3. **Make `score_word()` sensitive to rotatable characters:** Have `score_word()` calculate the `rotatable_ratio` of the individual token and apply a weirdness penalty if it exceeds `0.8`.

## Changes:

**1. CPU Pre-filter (Structural Fast-Check)**
Before any machine learning models run, a fast check identifies lines that don't need deep evaluation:

* **Empty:** Blank lines or pure whitespace.

* **Non-text:** Lines that are too short, lack unique symbols, have < 30% alphabetic characters, or match specific non-prose patterns (like Roman numerals or archive codes).

* *Exceptions:* Strings containing specific metadata markers (e.g., "Datum", "Tb.") or a high ratio of digits (e.g., measurements) bypass this filter to ensure they are processed.

**2. Immediate Overrides (First Match Wins)**
If a line reaches the ML pipeline, it is checked against a strict priority list where the first triggered condition assigns the category, bypassing standard scoring:

* **Empty:** Structural blank (0 words).

* **Trash (Hard Sweep):** Extremely low language confidence (< 0.45) combined with high perplexity (> 1000.0).

* **Trash (Inverted Scan):** A line dominated by mirrored/rotated Czech function words without any upright Czech diacritics to save it.

* **Trash (Unreadable All-Caps):** All alphabetic words are uppercase but severely lack vowels (< 10%).

* **Trash (Garbage-Density):** Very high concentration of non-alphanumeric characters (≥ 35%).

* **Trash (Short-Garbage):** 3 words or fewer, no Czech diacritics, low language score, and contains structural weirdness or gibberish.

* **Clear (High LM Confidence):** Language model perplexity is very low (< 50.0) on a line with 3 or more words.

**3. Quality Score Thresholds**
If no overrides catch the line, it relies on a dynamically weighted **Quality Score (0 to 1)**.

* **Score < 0.55** ➔ **Trash**.

* **Score < 0.85** ➔ **Noisy**.

* **Score ≥ 0.85** ➔ **Clear**.

**4. Secondary Checks & Band Guards**

* **Near-Boundary Promotion (Override 4):** A line rated `Noisy` (score 0.65–0.84) can be promoted to `Clear` if it is at least 4 words long, lacks structurally "weird" words, and has good LM perplexity.

* **Mostly Readable Cap:** A line mathematically qualifying for `Clear` is capped at `Noisy` if its ratio of valid structural words drops below 85%.

* **Short-fragment Guard:** Prevents very short noisy fragments from slipping into `Clear`.

**5. Post-Processing Smoothing (Context Awareness)**
Finally, a document-level pass corrects anomalies that cannot be seen line-by-line:

* **Page Context:** Downgrades borderline `Noisy` lines to `Trash` on heavily garbage pages, and promotes high-scoring `Trash` to `Noisy` on predominantly clean pages.

* **Deduplication:** Standardizes repetitive headers/footers to their most frequent category.

* **Rolling Window:** Sinks isolated `Noisy` lines if they are completely surrounded by `Trash` lines.

* **Page-Level Inverted-Scan Sweep:** Downgrades clusters (or entire page majorities) of suspicious lines identified by a lack of diacritics or high rotational ambiguity.

### 1. The Vowelless Illusion (`WVL A` & `ClAŕ,`)

* **Why it survives:** The `score_word` function inherently trusts alphabetic characters. It sees `WVL` and `ClAŕ` as valid alphanumeric blocks, assigning them a high base score of `0.8500`.
* **The Fix:** We need a **phonetic anchor penalty**. Any word that is 3 or more characters long but contains absolutely zero vowels (and isn't a known abbreviation) is almost certainly OCR garbage. Adding a massive penalty for vowelless strings will immediately force `WVL A` into the `Trash`. We also need to increase the penalty weight for mid-uppercase characters to deal with strings like `ClAŕ`.

### 2. The Ledger / Table Loophole (`nonč, mI 47 žn dn A' 1074 484 "Nik 1726.`)

* **Why it survives:** This is classic table/ledger fragmentation. The pipeline evaluates the words individually. `47`, `1074`, and `1726` are valid numbers, and `nonč` looks like a valid Czech word. Because the individual pieces are mathematically "fine," the overall line achieves a `0.8500`.
* **The Fix:** We must introduce a **line-level fragmentation penalty**. If a line is heavily composed of numbers and disjointed 1-to-2 character tokens (e.g., `mI`, `žn`, `dn`, `A'`), it is contextual noise. By calculating the ratio of short/digit tokens to total tokens, we can detect ledger noise and artificially spike the weirdness score for the entire line.

### 3. The Symbol Cluster (`At . O/wvi`)

* **Why it survives as Noisy:** The previous fix for inverted text successfully caught the `w` in `O/wvi`, penalizing it down to `0.5500` (`Noisy`).
* **The Fix:** To push this the rest of the way into `Trash`, the vowelless penalty mentioned above will combine with the `w` penalty, completely tanking the score.



### 🟡 The `Noisy` Category: The Salvageable Mess

This category successfully captures fragmented text—lines that are bruised by bad OCR or structural formatting but still hold semantic value.

* **The Good Noise:** Lines like `"neschopný11 nebo „vymazati" ; dále"` or `"eni - trestá so pokutou penéžitou"` (punished by monetary fine). A Czech speaker can easily read through the injected numbers (`11`) or the slight vowel swaps (`so` instead of `se`, `penéžitou` instead of `peněžitou`) to understand the underlying text.
* **The Sneaky Leaks:** I do see strings like `"mos( Ku>gaaKa q>toqo \p9sA"` slipping into the `Noisy` bucket with a score of `0.5601`. To a native speaker, this isn't "noisy" Czech—it is entirely alien. It is upside-down text masquerading as words. This perfectly highlights why your recent `w/q/x` mathematical penalty is so necessary to push these the rest of the way into `Trash`.
