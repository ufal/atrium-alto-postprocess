## What landed + key findings (carry into the close-out write-up)

### B2 — DONE: the garbage constant is de-confounded
Implementation matches the plan exactly: `QS_GARBAGE_NORM_MAX = 0.35` added to config and declared
in `text_util_langID.py` (~`:158`); the **3 QS sites** (`compute_quality_score` half-weight
condition, `norm_garbage` scale, short-penalty) now use `QS_GARBAGE_NORM_MAX`; the **2 hard gates**
keep `CATEG_GARBAGE_DENSITY_HIGH`; registered in `_THRESHOLD_NAMES`, `SEARCH_SPACE`, `EDGE_PARAMS`,
`THRESHOLD_PARAMS`. Defaults equal ⇒ bit-identical ⇒ parity green.

**Finding (37-param sweep, permutation importance):** the hard gate `CATEG_GARBAGE_DENSITY_HIGH`
keeps importance **≈0.092**, while `QS_GARBAGE_NORM_MAX` collapses to **≈0.0016** (noise floor).
⇒ The garbage constant's leverage was always the hard `rule_garbage_density` gate, **never** the
quality-score normalization. The Part A confound is resolved, and the QS garbage-scale joins the
9 `QS_WEIGHT_*` as a **freezable, non-identifiable** parameter.

### B5 — DONE: no dead code; keep all 14 rules
Default-off `_fire()` / `rule_fire_capture()` instrumentation at the 14 action sites (parity-safe),
`tools/rule_coverage_report.py`, `tools/RULE_COVERAGE.md` (criterion + JSON schema + B1 hook),
`tests/test_rule_coverage.py`.

**Finding (full corpus):** **0 DEAD, 11 LOAD-BEARING, 3 REDUNDANT-HERE** (`rule_allcaps`,
`rule_garbage_density`, `rule_inverted`). No rule is unreachable ⇒ under the gold-free criterion,
**nothing is retireable**. The fire-counter earned its place: `run_ablation_study` mislabels those
3 structural guards "**PRUNE** (fully redundant)" (0 LOO flips), but coverage shows they **do fire**
(2 / 20 / 7×) — entangled, not dead. The conservative "keep the guards" stance is now evidence-backed.

## Remaining work

### Increment 1 (LEAD) — B3: model-free importance backend (Morris → Sobol via SALib)

**Why still worth doing despite conclusive-looking results:** the RF surrogate weakened after the
decouple — **oob_r² ≈ 0.684** (was ~0.81). Permutation importance from a moderately-fit surrogate
on 37 params / 19 docs deserves a **surrogate-free** confirmation. Morris/Sobol remove RF-MDI's
bias toward continuous params, and Sobol's **S1 vs ST** independently tests whether
`CATEG_GARBAGE_DENSITY_HIGH`'s 0.092 is genuine first-order sensitivity or interaction-driven —
the exact coupling question B2 was about.

Mirror the existing backend contract in `tools/const_importance_sweep.py`
(`run_sklearn_backend(*, data, base_constants, params, output_dir, n_trials, seed, metric,
direction, eval_kwargs)`), reusing `evaluate_dataframe`, `objective_value`, `coerce_constants`,
`normalize_importances`, `save_json`, `save_importance_plot`, and `SEARCH_SPACE[name]["low"/"high"]`.

1. `requirements-sweep.txt`: add **`SALib`** (pin a known version; SALib ≥1.4 moved to
   `SALib.sample.sobol` / `SALib.analyze.sobol` and the `ProblemSpec` API — pin to avoid drift).
2. Helper `_salib_problem(params)` →
   `{"num_vars": len(params), "names": list(params), "bounds": [[SEARCH_SPACE[n]["low"], SEARCH_SPACE[n]["high"]] for n in params]}`.
3. Helper `_constants_from_row(row, base_constants, params)` — like `sample_random_constants` but
   from a float array: int-cast for `spec["type"]=="int"`, then `coerce_constants`.
4. **Constraint handling** (the one real subtlety — SALib's structured matrices can't reject &
   resample the way the sklearn loop does at `:329`):
   - **Morris** screens **all 37**; *repair* the lone ordering constraint per row
     (`CATEG_NOISY_SCORE_MAX = max(CATEG_NOISY_SCORE_MAX, CATEG_TRASH_SCORE_MAX + 0.01)`) so every
     sample is valid; document the minor distortion on those two params.
   - **Sobol v1** restricts the variable set to the **constraint-free** constants (hold
     `CATEG_TRASH_SCORE_MAX` / `CATEG_NOISY_SCORE_MAX` at baseline) so S1/ST stay clean; document
     the limit (a constrained sampler is future work).
5. `run_morris_backend(...)`: `morris.sample(problem, N=morris_r, num_levels=4)` (≈`r·(D+1)` evals)
   → evaluate each row via `evaluate_dataframe`→`objective_value` → `morris.analyze(...)`. Save
   normalized **`mu_star`** as `param_importance.json` (+ raw `mu_star`/`sigma`/`mu`); `sigma` flags
   interactions; reuse `save_importance_plot`.
6. `run_sobol_backend(...)`: `saltelli.sample(problem, sobol_n, calc_second_order=False)`
   (≈`N·(D+2)` evals) → evaluate → `sobol.analyze(..., calc_second_order=False)` → save **S1** and
   **ST**; importance = normalized ST. **Expensive** (e.g. N=256, D=35 ⇒ ~9.5k evals) → a cluster
   job; Morris is the cheap laptop screen.
7. CLI: extend `--backend` choices (`:543`) with `morris`, `sobol`; add `--morris-r` (default 10)
   and `--sobol-n` (default 256); dispatch in `main()` (~`:622`).
8. **Must keep stating in output + docs:** Morris/Sobol still score against **self-generated
   labels** — a better *importance/coupling* tool, **not** a correctness tool.

**Files:** `tools/const_importance_sweep.py`, `requirements-sweep.txt`,
`tests/test_sweep_backends.py` (new — tiny-budget smoke asserting `mu_star`/`sigma` for morris and
`S1`/`ST` for sobol on the fixture).
