# Strategy — Further development of issue #5 (config-constant importance)

## Context

Issue [#5](https://github.com/ufal/atrium-alto-postprocess/issues/5) built a lightweight
surrogate that ranks which `[CLASSIFY]`/`[TEXT_UTILS]` config constants actually move line
categorization, using the per-line `DOC_LINE_CATEG` CSVs as immutable ground truth and replaying
the *real* production engine (`compute_quality_score` → `determine_category`/`categorize_line` →
`apply_document_postprocessing`) under overridden constants. The research arc is complete and the
findings are solid as a **sensitivity map**, but with one load-bearing caveat: the CSV `categ`
labels are produced by the current config, so every objective is maximized at the current config
**by construction** (baseline `flip_rate == 0`). The tooling answers *"which constants can change
the categorization"* — not *"which config is objectively better"*. That requires gold labels.

This plan turns the digest's Part B (B1–B6) into an executable development strategy. Per the
user's scoping decisions:
- **Deliverable** = phased roadmap (B1–B6) **+** an implementation-ready spec for the first increment.
- **First increment leads with B5** (rule cleanup / coverage-gated retirement).
- **B1 (gold labels) is deferred** — kept as a documented future direction; near-term work stays
  in the offline / self-referential tooling lane (B2/B3/B4/B5).

**Intended outcome:** ship safe, reusable offline tooling and provably-justified code cleanup
*without* claiming a recalibration the data can't support, and leave a clear runway to break the
self-reference later if a gold set becomes available.

## The strategy (phased roadmap)

Ordered by what is *realizable now* under the no-gold constraint, with B5 leading per the user.

### Phase 1 — Offline tooling & cleanup (no humans, parity-safe). Realizable now.
- **B5 (FIRST INCREMENT, this plan):** rule-fire **coverage instrumentation** + a coverage report
  tool, so rule retirement can be justified by *"never fires across collections"* (a
  config-independent, gold-free criterion) instead of *"redundant on the frozen self-labels."*
- **B2:** decouple the overloaded `CATEG_GARBAGE_DENSITY_HIGH` into a separate
  `QS_GARBAGE_NORM_MAX` (parity-preserving) so the hard gate and the quality-score scale can be
  A/B'd independently — removes the confound that makes the 0.55 result un-trustworthy.
- **B3:** add a model-free, variance-based importance backend (Morris screen → Sobol) via SALib;
  removes RF-MDI's bias toward continuous params and exposes coupling (e.g. the garbage three-way
  reuse).
- **B4 (finish):** add a cross-backend **agreement/consensus** check; always report per-document;
  run the real studies on the full corpus. (`evaluate_per_document` already exists.)

### Phase 2 — Break the self-reference (needs a human gold set). Deferred.
- **B1:** `tools/build_label_queue.py` active-learning extractor (flip set, decision-margin,
  rule-boundary, high-disagreement lines) + an optional `gold_column` path in `evaluate_dataframe`
  that scores only labeled rows against the human label. **Not implemented now** (gold out of
  scope); the hook points are documented below so it can land cleanly later.
- **B6:** once a gold set exists, optionally replace the threshold cascade with a small
  **monotonic/interpretable** classifier (monotonic GBM / calibrated logistic) over the same
  precomputed features, with a cost-sensitive objective (Clear→Trash loss ≫ Trash→Noisy leak).
  Documented future direction only.

**Dependency line:** B5 ⟂ B2 ⟂ B3 (independent, any order; B5 first per user). B4-consensus
benefits from B3 existing. B1 unblocks the *correctness* interpretation of B2/B3/B5 and is the
prerequisite for B6.

## First increment — B5: rule-fire coverage instrumentation (implementation-ready)

**Problem it solves.** The current ablation tools (`run_ablation_study.py`,
`greedy_backward_elimination.py`) measure a rule's **decisive** effect = lines that flip vs. the
frozen baseline when the rule is disabled (LOO). On the self-labeled corpus this conflates two very
different things and is blind to **survivor entanglement** (two overlapping rules each show 0
flips alone because the other catches the line — yet neither is dead). What's missing is the raw
signal: **does the rule's action branch ever actually execute?** A rule whose action never fires
across *all* collections is genuinely unreachable dead code — safe to retire *without* gold labels,
because deletion provably changes nothing. That is the only honest gold-free retirement criterion.

### Step 1 — Add default-off fire instrumentation to the engine (`text_util_langID.py`)
Mirror the existing `DISABLED_RULES` pattern (module global at `:29`). Add:
```python
RULE_FIRE_COUNTS: dict | None = None   # None => production, zero overhead, byte-identical
def _fire(name: str) -> None:
    if RULE_FIRE_COUNTS is not None:
        RULE_FIRE_COUNTS[name] = RULE_FIRE_COUNTS.get(name, 0) + 1
```
Call `_fire("rule_x")` at the exact point each rule **commits its action** (the branch that returns
a category or applies a penalty), at the 14 sites already gated by `DISABLED_RULES`:
- `determine_category` (`:921–:992`): `rule_hard_sweep`, `rule_extreme_ppl`, `rule_absolute_ppl`,
  `rule_inverted`, `rule_allcaps`, `rule_garbage_density`, `rule_trailing_fill_rescue`,
  `rule_short_garbage`, `rule_lowppl_clear`, `rule_mostly_readable_noisy` — record just before each
  `return`.
- `categorize_line` penalty block (`:1014`, `:1020`, `:1026`, `:1033`): `penalty_wqx_rot`,
  `penalty_vowelless`, `penalty_ledger_fragmentation`, `penalty_mid_uppercase` — record where the
  penalty is applied.

Because `determine_category` short-circuits on the first matching rule, `_fire` records the rule
that was *actually decisive in production order* — exactly the contribution we want to count.
Default `None` ⇒ the `is not None` guard makes production behavior identical; **the existing
`tests/test_recategorize_parity.py` proves parity is preserved**.

Add a small context manager for clean capture/reset:
```python
@contextmanager
def rule_fire_capture():
    global RULE_FIRE_COUNTS
    prev, RULE_FIRE_COUNTS = RULE_FIRE_COUNTS, {}
    try:
        yield RULE_FIRE_COUNTS
    finally:
        RULE_FIRE_COUNTS = prev
```

### Step 2 — New tool `tools/rule_coverage_report.py`
Reuses `recategorize_from_csv` (`load_csvs`, `recategorize_dataframe`, `evaluate_dataframe`,
`read_config_constants`, `_load_lang_config`) and `text_util_langID.rule_fire_capture` +
`override_constants` — **no parallel re-implementation** (consistent with SWEEP_NOTES "one engine,
not two"). For each collection dir and each of the 14 rules it reports:

| Column                     | Source                                                                      | Meaning                                                   |
|----------------------------|-----------------------------------------------------------------------------|-----------------------------------------------------------|
| `fire_count` / `fire_rate` | `rule_fire_capture` over a single recategorize pass                         | times the rule's action executed (overlap-independent)    |
| `decisive_count`           | LOO: `evaluate_dataframe` with `DISABLED_RULES={rule}`, `flip_rate × n`     | lines whose final category changes if the rule is removed |
| `clear_loss`               | `confusion["Clear"]→Trash/Non-text` (helper already in both ablation tools) | valid text destroyed if removed                           |
| `class`                    | derived                                                                     | DEAD / REDUNDANT-HERE / LOAD-BEARING                      |

Classification:
- `fire_count == 0` across **all** collections → **DEAD** (unreachable) → retire candidate.
- `fire_count > 0` and `decisive_count == 0` → **REDUNDANT-HERE** (fires but currently masked by an
  overlapping rule) → **keep** (a real guard, exposed by order/corpus changes).
- `decisive_count > 0` → **LOAD-BEARING** → keep.

CLI accepts multiple `--input-dir` and aggregates; emits `rule_coverage.json` + a Markdown table.

### Step 3 — Retirement criterion & docs (no blind deletion)
Retire a rule **only** when it is `fire_count == 0` aggregated across the **full multi-collection
corpus**, and never retire the cheap structural guards (`rule_inverted`, `rule_allcaps`,
`rule_garbage_density`) unless they are coverage-empty across a broad, explicitly-approved
collection set (per digest Part A — they cost ~nothing and guard failure modes absent from small
samples). Document the criterion + run recipe in `tools/SWEEP_NOTES.md` (or a new
`tools/RULE_COVERAGE.md`). **The actual rule deletions are a follow-up commit that lands only after
the full-corpus coverage run confirms `fire_count == 0`** — produced on the cluster (or here if the
full corpus is provided). This increment ships the instrument + criterion + smoke validation, not
speculative deletions.

### Step 4 — Tests
- Extend `tests/test_recategorize_parity.py` (or a new `tests/test_rule_coverage.py`): with
  `RULE_FIRE_COUNTS = None` the engine output is unchanged (parity); inside `rule_fire_capture()` a
  line known to trip a specific rule increments exactly that rule; the sum of decisive
  `determine_category` fires equals the number of non-`Empty` lines routed by a rule branch.
- Smoke-run `tools/rule_coverage_report.py` on `data_samples/DOC_LINE_CATEG` end-to-end.

**Critical files (first increment):** `text_util_langID.py` (instrumentation),
`tools/rule_coverage_report.py` (new), `tools/SWEEP_NOTES.md` (criterion),
`tests/test_rule_coverage.py` (new) / `tests/test_recategorize_parity.py`.

## Roadmap detail (Phase 1 remainder + Phase 2) — hook points for later increments

### B2 — Decouple `CATEG_GARBAGE_DENSITY_HIGH` (cheap, parity-preserving)
The constant is read once (`text_util_langID.py:115`) and used at **5 sites**: 2 hard gates
(`:897` trailing-fill rescue, `:952` `rule_garbage_density` gate) and 3 quality-score sites
(`:1193` short half-weight, `:1196` `norm_garbage` scale, `:1229` short penalty). Plan:
1. `config_langID.txt` `[TEXT_UTILS]`: add `QS_GARBAGE_NORM_MAX = 0.35` (next to `:64`).
2. `text_util_langID.py` near `:115`: `QS_GARBAGE_NORM_MAX = _get_float("TEXT_UTILS", "QS_GARBAGE_NORM_MAX", 0.35)`.
3. Replace the constant with `QS_GARBAGE_NORM_MAX` at the **3 QS sites only** (`:1193`, `:1196`,
   `:1229`); leave `CATEG_GARBAGE_DENSITY_HIGH` at the 2 hard gates. Both default `0.35` ⇒
   **bit-identical** ⇒ parity test stays green.
4. Register `QS_GARBAGE_NORM_MAX` in `recategorize_from_csv.py` `_THRESHOLD_NAMES` (`:394`) **and**
   `const_importance_sweep.py` `SEARCH_SPACE` (`:93`, range `{float, 0.15, 0.60}`) — the `_missing`
   guard (`:142`) *requires* this or the sweep raises. Optionally add to `EDGE_PARAMS`/`THRESHOLD_PARAMS`.
   → sweep becomes **37 params**; the garbage gate and QS scale are now independently A/B-able.
5. Test: defaults reproduce identical categories; a divergence case where `QS_GARBAGE_NORM_MAX ≠
   CATEG_GARBAGE_DENSITY_HIGH` moves the score but not the hard gate.

### B3 — Model-free importance backend (Morris → Sobol) via SALib
- Add `salib` to `requirements-sweep.txt`; extend `--backend` choices (`const_importance_sweep.py:543`)
  with `morris`, `sobol`; add `run_morris_backend` / `run_sobol_backend` mirroring
  `run_sklearn_backend`'s signature/outputs.
- **Morris** (`r·(D+1)` evals): rank by `mu_star`, report `sigma` (interaction flag) — cheap screen.
- **Sobol** (`N·(2D+2)`, Saltelli): first-order `S1` vs total `ST`. v1 **restricts to
  constraint-free constants** (Saltelli's structured `(A,B,ABᵢ)` matrix breaks ordering constraints
  like `CATEG_TRASH_SCORE_MAX < CATEG_NOISY_SCORE_MAX`); document the limit.
- **Must keep stating:** still scored against self-generated labels — a better *importance/coupling*
  tool, **not** a correctness tool. `ST ≫ S1` on the garbage constant is the quantitative evidence
  for the B2 decoupling.

### B4 — Cross-backend consensus + full-corpus discipline (mostly done)
- `evaluate_per_document` and `baseline_per_document.json` already exist. Remaining: a small
  `tools/importance_consensus.py` (or `--consensus` flag) that loads `param_importance.json` from
  sklearn/optuna/morris/sobol runs and flags a ranking "stable" only when the top-K broadly agree
  (top-K overlap / rank correlation). Always run the real studies on the full corpus.

### B1 — Gold set (DEFERRED; documented hook points)
- `evaluate_dataframe` (`recategorize_from_csv.py:710`) gains an optional `gold_column`; when set,
  `_metrics_from_labels` (`:678`) scores **only labeled rows** against the human label instead of
  the self-generated `categ`. An all-current-labels gold file must reproduce the self-referential
  numbers exactly (sanity check).
- `tools/build_label_queue.py` emits a focused human-labeling queue (A/B flip set, decision-margin
  lines near `CATEG_*_SCORE_MAX`, rule-boundary lines from the B5 instrumentation, high-disagreement
  lines across trials) with `n_flips`, `alt_categories`, `margin`, empty `gold_categ`.
- **Not built now** — this is the work that converts "what changes the labels" into "what is
  better." Revisit when labeling capacity exists.

### B6 — Interpretable model (DEFERRED, needs B1)
Once gold exists, optionally swap the hand-tuned cascade for a monotonic GBM / calibrated logistic
over the same precomputed features, with an asymmetric cost (Clear→Trash ≫ Trash→Noisy). Ends the
"threshold treadmill"; keeps auditability. Documented direction only.

## Risks / explicit non-goals
- **No recalibration claims.** Nothing here proves a config is *better*; without gold labels every
  numeric recommendation (incl. `CATEG_GARBAGE_DENSITY_HIGH 0.35→0.55`) stays a *reviewed candidate*,
  not a production change.
- **No blind rule deletion.** B5 ships the instrument + criterion; deletions wait on the full-corpus
  coverage run. Structural guards stay regardless on small samples.
- **Engine hot-path touch (B5).** Instrumentation is default-off and parity-locked; the only cost is
  one `is not None` check per rule when disabled.
- **Sobol constraint limit (B3).** v1 covers constraint-free constants only; the ordered category
  thresholds need a constrained sampler (future).
