## Calibration methodology (make the next pass reproducible)

### 1 — Fixed evaluation harness
- **Replay, don't rerun.** All calibration uses `recategorize_from_csv.py` over the immutable
  per-line CSVs (GPU perplexity + FastText language scores are pre-computed → cheap sweeps, no
  GPU). Any config edit is evaluated by replaying the same baseline CSVs (`0.75` / `0.50`
  versions are the agreed foundation).
- **Parity gate.** Every change must keep `tests/test_recategorize_parity.py` green: at the
  default config the replay reproduces stored `categ` (`flip_rate == 0`). Calibration changes
  the *config*, not the replay contract.

### 2 — Evaluation metrics (the exact set)
- **Primary:** per-line **confusion matrix** + **macro-F1** across the five categories (the
  #5/B5 finding showed a one-directional cost proxy hides Trash→Noisy leakage — use the true
  confusion matrix, not `min(clear_drop, trash_rise)`).
- **Boundary-focused:** confusion specifically on **Noisy↔Clear**, **Noisy↔Trash**,
  **Non-text↔Trash** (the three disagreement zones from #2's expert review).
- **Marginal-case count:** number and identity of lines within a small QS margin of a
  boundary (the "8 of 1348" metric) — the practical review-burden signal.
- **Stability:** flip-rate vs the previous frozen config (how many lines move category per
  threshold change).

### 3 — Review workflow with the domain experts
- Produce a small **review CSV** of only the marginal + flipped lines (not the whole corpus)
  for DanaKriv / ARUP-ARUB, with the per-rule diagnostic boolean columns already added so the
  reviewer sees *why* a line landed where it did.
- Capture expert verdicts as a candidate **gold column** — this is the bridge to #5/B1 and the
  only way to convert "deviates from self-generated labels" into "objectively right/wrong".

## Open edge cases to resolve (flagged "confirm with a Czech speaker")
| Case | Example | Proposed fix |
|---|---|---|
| **Vowelless illusion** | `WVL A`, `ClAŕ,` survive as Clear (qs≈0.85) | phonetic-anchor penalty for 3+ char zero-vowel non-abbreviations + heavier mid-uppercase penalty |
| **Ledger / table loophole** | `nonč, mI 47 žn dn A' 1074 …` ≈ 0.85 | line-level fragmentation penalty (ratio of short/digit tokens spikes weirdness) |
| **Symbol cluster** | `At . O/wvi` sits at Noisy | combine vowelless penalty with existing `w` penalty → push to Trash |
| Collateral (working as intended) | `Maždý…` Clear→Noisy; `P.T. panu` Noisy→Trash | correct QS behaviour, **not** regressions — leave |

## Verification / done-criteria
- The three open edge cases are confirmed (or consciously accepted) by a Czech speaker.
- A documented calibration run reproduces the headline numbers on the **full** corpus (not the
  smoke fixture), with the confusion-matrix + macro-F1 harness above.
- Parity test green; the frozen config and its flip-rate-vs-previous are recorded.
- The marginal/flipped review CSV is captured as candidate gold (feeds #5/B1).
