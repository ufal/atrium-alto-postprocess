# Strategy — Issue #3: Calibration of the categorization logic

> Issue [#3](https://github.com/ufal/atrium-alto-postprocess/issues/3) · Labels: `enhancement`, `development` · **Status: ✅ 2026-07-02 pass shipped in `v0.20.1` (2026-07-03, `test`→`main` synced) — awaiting final ARUP/ARUB review of the shared `307` archives**

## Context
#3 is the **calibration loop** on top of the #2 definitions: an iterative cycle with
**DanaKriv** (and the ARUP/ARUB teams) reviewing real CSV outputs while K4TEL tunes the
detectors, the quality-score weights, and the category boundaries. By 2026-06-19 the engine
was frozen for review at **8 of 1348 lines (0.6%)** marginal; `recategorize_from_csv.py`
replays categorisation offline after a config edit. A 2026-06-28 parameter-importance sweep
(issue #5) found **11 LOAD-BEARING / 3 REDUNDANT-HERE / 0 DEAD** rules on the live dataset.

The **2026-07-02 review with Dana Kriv** produced four concrete false-negative findings on
real data (verified against `CTX199603106`, a Charles-Bridge archaeological report), plus one
"Extra" item:

1. **Short domain abbreviations** — `mm`, `cm`, `Tb.`, `tb.`, `č.neg.`, `č, neg.` (+ comma/case
   variants) are dumped to **Trash** (or **Non-text**); they should be **Noisy**.
   Units glued to digits (`14mm`, `6,5 cm`) are already correct and must stay correct.
2. **Short numbered headlines / captions** land in **Trash** — a real word or two surrounded by
   digits/symbols drags `valid_word_ratio` (→ `quality_score`) down. Real examples:
   `2, Popis nálezu i - 3`, `4. Literatura 5`, `7, Plánky 1 - 2`, `Plánek č. 1`. Should be **Noisy**.
3. **All-caps single-word headlines** (e.g. `LITERATURA`) are marked **Non-text**; they should
   be scored normally (Clear/Noisy).
4. The **Noisy↔Clear** quality-score boundary should drop **0.85 → 0.80**.
5. **Extra**: the remapped lang score should be **always** 0.75 or 0.5 — *"the original lang
   score should not matter."*

Plus: *"expansion of test set fixtures is welcome."*

This plan defines the changes for this pass **and** keeps the reproducible calibration
methodology (replay + parity gate + review CSV) established in the previous version.

## Calibration methodology (unchanged — the next pass stays reproducible)

### 1 — Fixed evaluation harness
- **Replay, don't rerun.** Calibration uses `recategorize_from_csv.py` over the immutable
  per-line CSVs (perplexity + FastText scores are frozen in the CSV → no GPU). `_rescore_row`
  (`tools/recategorize_from_csv.py:206,220`) calls the **real** `compute_quality_score` /
  `categorize_line`, so new rules are exercised automatically — no logic is mirrored.
- **Parity gate.** `tests/test_recategorize_parity.py` must stay green: at the *default* config
  the replay reproduces stored `categ` (`flip_rate == 0`). A calibration pass changes the
  config/rules **and** re-baselines the stored `categ`; the hand-curated calibration fixtures
  (below) are the real semantic guard against wrong flips.

### 2 — Evaluation metrics (unchanged)
Per-line **confusion matrix + macro-F1**; boundary-focused confusion on **Noisy↔Clear /
Noisy↔Trash / Non-text↔Trash**; **marginal-case count** (the "8/1348" signal); **flip-rate**
vs the previous frozen config.

### 3 — Review workflow
Produce a **review CSV of only the flipped + marginal lines** (with the per-rule diagnostic
boolean columns) for DanaKriv / ARUP-ARUB, and capture expert verdicts as a candidate **gold
column** (bridge to #5/B1).

## Calibration pass — 2026-07-02 (DanaKriv notes)

All rules/thresholds live in **`text_util_langID.py`** and are config-overridable from the
`[TEXT_UTILS]` section of **`config_langID.txt`**; the in-code default is kept equal to config.
Category is a first-match cascade in `determine_category()` (`text_util_langID.py:1094`),
tail-routing on QS bands `<0.55 Trash / [0.55,0.80) Noisy / ≥0.80 Clear`.

| #   | Change                                           | Where                                                                                                      | Mechanism                                                                    |
|-----|--------------------------------------------------|------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------|
| 4   | Noisy↔Clear `0.85 → 0.80`                        | `config_langID.txt:61` + `text_util_langID.py:151` (`CATEG_NOISY_SCORE_MAX`)                               | band alignment (`categorize_line`) follows the constant automatically        |
| 3   | all-caps word not Non-text                       | `is_non_text()` `text_util_langID.py:1426` onward                                                          | vowel-ratio guard on the single-token branch                                 |
| 1+2 | abbreviations & numbered headlines → floor Noisy | `is_forgiven_headline()` (`:988`); hooked into `determine_category` (`:1175`) + `pre_filter_line` (`:740`) | tight token classifier, lifts Trash→Noisy without demoting Clear             |
| 1   | route bare/short abbrev lines to scoring         | `config_langID.txt:39` (`METADATA_MARKERS`) + `SHORT_EXCEPTION_TOKENS` (`config_langID.txt:100`)           | escape the `<4 chars`/Non-text pre-filter gates                              |
| 5   | Extra — remap score unconditional, toggleable    | `remap_lang()` `text_util_langID.py:507`; new `LANG_REMAP_ALWAYS` (`config_langID.txt:87`)                 | boolean config gate; `true`=unconditional 0.75/0.50, `false`=prior #3 A1 cap |

### Change 4 — boundary 0.85 → 0.80
Set `CATEG_NOISY_SCORE_MAX = 0.80` in config and code default. Lines with QS ∈ [0.80, 0.85)
flip Noisy→Clear. Calibration `NOISY` fixtures only assert `!= Trash`, so they stay green;
`CLEAR` fixtures are only helped. **Locked** by `TestCategorizeLineReason.test_clear_threshold_with_clamped_score`
in `tests/test_text_utils.py`, expressed relative to the constant (not a hardcoded `0.82`).

### Change 3 — all-caps headline word → scored, not Non-text
The single-token branch of `is_non_text` returns `Non-text` for any all-caps token with
`len ≥ 10` or containing `X` — exactly what traps `LITERATURA`. Added a guard: a single
**alphabetic** all-caps token whose `compute_vowel_ratio(tok) ≥ VOWEL_RATIO_LOW` (a real
headline word) returns `False` → `Process` → scored. Digit-bearing tokens and vowel-starved
all-caps (codes/garbage) stay `Non-text`; anything containing `X` still does too. Verified
against adversarial cases (`XXXXXXXXXX`, `CTX199603106`-style codes, short vowel-starved
tokens below the `len≥10` gate) — all behave as intended. Multi-word all-caps headers already
passed the pre-filter — unchanged.

### Changes 1 & 2 — "forgiven headline / abbreviation" floor
`is_forgiven_headline(text, garbage_density)` classifies each token as NUMBERING (digit ≤
`HEADLINE_MAX_DIGITS`, or roman numeral) / ABBREV (`SHORT_EXCEPTION_TOKENS`, `ACADEMIC_TITLES`,
or a `METADATA_MARKERS` marker — supplies both content and context on its own) / FUNCTION
(`SHORT_VALID_WORDS`/`SINGLE_CHAR_ALLOWED`) / CLEAN WORD (multi-token lines only — a bare
"clean-looking" single word is exactly the profile of an inverted-scan/short-garbage token) /
STRUCTURAL (pure punctuation) / GARBAGE (anything else, disqualifies the line). A line is
forgiven only when it has both real **content** and genuine numbering/abbreviation **context**
— this is what keeps a bare short prose fragment (no numbering, no abbreviation) out of the
rescue path.

Hooked into `determine_category` (computed once, after hard-sweep/inverted/all-caps/
garbage-density — so genuine garbage is untouched) and `pre_filter_line` (after
`METADATA_MARKERS`, before the Non-text gates). New rule `rule_forgiven_headline`; rule
coverage count is **15** (`tests/test_rule_coverage.py`, dynamically computed from
`tools/rule_coverage_report.py:RULES` — no hardcoded bump needed).

New config (`[TEXT_UTILS]`): `SHORT_EXCEPTION_TOKENS`, `HEADLINE_MAX_WORDS = 8`, and
`HEADLINE_MAX_DIGITS = 2` (an addition beyond the original ask — pure-digit tokens longer than
this are GARBAGE, not structural numbering, keeping archive/stamp codes like `6694` from being
swept up as forgiven captions).

**Deviation from the original plan text, verified deliberate:** `METADATA_MARKERS` was **not**
extended with the comma/spacing variants (`č,neg`, `Tb,`, etc.) originally proposed. Direct
testing confirmed `is_forgiven_headline`'s token-level, punctuation-normalizing classification
already covers every DanaKriv variant on its own — extending the crude line-wide substring
match would have been redundant and slightly less precise. All target examples verified
resolving to `Process`/`Noisy` without the extension.

### Change 5 — Extra: remapped lang score, always 0.75/0.5 (toggleable)
`remap_lang()` previously used `min(score, cap)` — a **ceiling** (`#3 A1`, an earlier pass
addressing "a weak guess must not be inflated"). DanaKriv's 2026-07-02 note explicitly reverses
that, restating an ask from earlier in the thread ("the languages, except for trusted, should
have score 0.75") — the original score must **not** matter.

Rather than hard-code the reversal (this exact call has now flipped twice), added a boolean
config switch, **`LANG_REMAP_ALWAYS`** (`config_langID.txt:87`, default `true`):
- `true`: `remap_lang()` unconditionally assigns `LANG_SCORE_REMAP` (0.75, Latin-script guesses)
  or `LANG_SCORE_REMAP_FAR` (0.50, non-Latin) to every remapped (non-trusted, non-`slk`) row.
- `false`: restores the `#3 A1` ceiling — the fixed value is only assigned when the original
  score exceeds it; a weaker original guess is left untouched.

`slk` keeps its own score-preservation branch, untouched (pre-existing, unrelated to this note).
The unremapped `orig_lang_score` is untouched either way and still drives the hard-sweep /
`rule_wqx_rot` / `rule_vowelless` gates in `determine_category` — this only affects the
`QS_WEIGHT_LANG` component (5% weight) and the stored `lang_score` CSV column.

**Aside, not fixed (out of scope):** while tracing this, found `remap_lang`'s `if base ==
"slk": return new_label, score` branch is dead code under the live config — `slk` is already
in `TRUSTED_FOREIGN_LANGS`, so it's caught by the `known_bases` branch first (score preserved
either way; only the label differs — stays `slk_Latn` instead of `ces_Latn`). Cosmetic, unrelated
to this pass, flagged for awareness only.

Expected flips on the review lines: `2, Popis nálezu i - 3`, `4. Literatura 5`,
`7, Plánky 1 - 2`, `Plánek č. 1` → Trash→**Noisy**; `mm`/`cm`/`č.neg.` → Non-text/Trash→**Noisy**;
`LITERATURA` → Non-text→**scored**; clean prose and confirmed garbage unchanged.

## Session fixes — 2026-07-03 verification pass

Before this pass could be trusted, two implementation gaps were found by running the real test
suite and replaying the actual DanaKriv fixture lines through the engine (not just reading code):

**Bug 1 — dead-code duplicate silently defeated the rescue.** `determine_category` had *two*
copies of the `rule_wqx_rot` / `rule_vowelless` / `rule_ledger_fragmentation` /
`rule_mid_uppercase` checks: an old one immediately after rule 4 (unconditional early `return`,
no knowledge of `forgiven`), and a correctly-ordered one right before QS-band routing (calls
`check_rescues()`, which does consult `forgiven`). The first always returned before the second
could run, making the "moved down" copy permanently unreachable. Confirmed on real fixture
data: `mm`, `2, Popis nálezu i - 3`, and `7, Plánky 1 - 2` all mis-routed to `Trash` under the
unpatched engine. **Fix:** removed the dead top copy (`text_util_langID.py:1116` area);
the single authoritative check now lives at `:1198`. Zero flip on the local sample corpus after
the fix — pure bug removal, no side effects.

**Bug 2 — new fixture groups never wired into a test.** `HEADLINE_NUMBERED`, `SHORT_EXCEPTIONS`,
`ALLCAPS_HEADLINE` existed in `tests/calibration_fixtures.py` but no test function imported
them — which is exactly why Bug 1 wasn't caught earlier. **Fix:** added the three test
functions to `tests/test_calibration.py` (confirmed they fail against the pre-Bug-1-fix engine
using the real frozen fixture values, and pass after the fix — genuine regression guards, not
just green noise).

**Real-corpus validation** (`tools/recategorize_from_csv.py data_samples/DOC_LINE_CATEG
--report-only`, run locally by K4TEL after applying the reviewed files): **20 lines flipped
Trash→Noisy, zero collateral movement elsewhere.** `CTX199603106` shows the exact four named
target lines flipping; `CTX193001369` shows 7 `Tb.`/`č.neg.` abbreviation flips; 9 further
lines flip on unlisted real cases (`CESKOSLOVENSKA AKADEM'E VEk`, `hrad sbrč.2`, etc.),
suggesting the fix generalizes rather than overfitting to the named examples.
