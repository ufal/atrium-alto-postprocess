## Context
#3 is the **calibration loop** on top of the #2 definitions: an iterative cycle with
**DanaKriv** (and the ARUP/ARUB teams) reviewing real CSV outputs while K4TEL tunes the
detectors, the quality-score weights, and the category boundaries. By 2026-06-19 the engine
was frozen for review at **8 of 1348 lines (0.6%)** marginal; `recategorize_from_csv.py`
replays categorisation offline after a config edit.

The **2026-07-02 review with Dana Kriv** produced four concrete false-negative findings on
real data (verified against `CTX199603106`, a Charles-Bridge archaeological report):

1. **Short domain abbreviations** — `mm`, `cm`, `Tb.`, `tb.`, `č.neg.`, `č, neg.` (+ comma/case
   variants) are dumped to **Trash** (or **Non-text**); they should be **Noisy**.
   Units glued to digits (`14mm`, `6,5 cm`) are already correct and must stay correct.
2. **Short numbered headlines / captions** land in **Trash** — a real word or two surrounded by
   digits/symbols drags `valid_word_ratio` (→ `quality_score`) down. Real examples:
   `2, Popis nálezu i - 3`, `4. Literatura 5`, `7, Plánky 1 - 2`, `Plánek č. 1`. Should be **Noisy**.
3. **All-caps single-word headlines** (e.g. `LITERATURA`) are marked **Non-text**; they should
   be scored normally (Clear/Noisy).
4. The **Noisy↔Clear** quality-score boundary should drop **0.85 → 0.80**.

Plus: *"expansion of test set fixtures is welcome."*

This plan defines the changes for this pass **and** keeps the reproducible calibration
methodology (replay + parity gate + review CSV) established in the previous version.

> **Delivery for this pass:** refined source files are returned **in chat** for review; **no
> pushes to remote branches**. The engine change is config-driven and parity-gated so it stays

## Calibration methodology (unchanged — the next pass stays reproducible)

### 1 — Fixed evaluation harness
- **Replay, don't rerun.** Calibration uses `recategorize_from_csv.py` over the immutable
  per-line CSVs (perplexity + FastText scores are frozen in the CSV → no GPU). `_rescore_row`
  (`tools/recategorize_from_csv.py:206,220`) calls the **real** `compute_quality_score` /
  `categorize_line`, so new rules are exercised automatically — no logic is mirrored.
- **Parity gate.** `tests/test_recategorize_parity.py` must stay green: at the *default* config
  the replay reproduces stored `categ` (`flip_rate == 0`). A calibration pass changes the
  config/rules **and** re-baselines the stored `categ`; the hand-curated calibration fixtures
  (below) are the real semantic guard against wrong flips.

### 2 — Evaluation metrics (unchanged)
Per-line **confusion matrix + macro-F1**; boundary-focused confusion on **Noisy↔Clear /
Noisy↔Trash / Non-text↔Trash**; **marginal-case count** (the "8/1348" signal); **flip-rate**
vs the previous frozen config.

### 3 — Review workflow
Produce a **review CSV of only the flipped + marginal lines** (with the per-rule diagnostic
boolean columns) for DanaKriv / ARUP-ARUB, and capture expert verdicts as a candidate **gold
column** (bridge to #5/B1).

## Calibration pass — 2026-07-02 (DanaKriv notes)

All rules/thresholds live in **`text_util_langID.py`** and are config-overridable from the
`[TEXT_UTILS]` section of **`config_langID.txt`**; the in-code default is kept equal to config.
Category is a first-match cascade in `determine_category()` (`:947-1047`), tail-routing on QS
bands `<0.55 Trash / [0.55,0.85) Noisy / ≥0.85 Clear`.

| #   | Change                                           | Where                                                                              | Mechanism                                                        |
|-----|--------------------------------------------------|------------------------------------------------------------------------------------|------------------------------------------------------------------|
| 4   | Noisy↔Clear `0.85 → 0.80`                        | `config_langID.txt:61` + `text_util_langID.py:151` (`CATEG_NOISY_SCORE_MAX`)       | band alignment (`:1117-1119`) follows the constant automatically |
| 3   | all-caps word not Non-text                       | `is_non_text()` `text_util_langID.py:1183-1185`                                    | vowel-ratio guard on the single-token branch                     |
| 1+2 | abbreviations & numbered headlines → floor Noisy | new `is_forgiven_headline()`; hooked into `determine_category` + `pre_filter_line` | tight token classifier, lifts Trash→Noisy without demoting Clear |
| 1   | route bare/short abbrev lines to scoring         | `config_langID.txt:39` (`METADATA_MARKERS`) + new `SHORT_EXCEPTION_TOKENS`         | escape the `<4 chars`/Non-text pre-filter gates                  |

### Change 4 — boundary 0.85 → 0.80
Set `CATEG_NOISY_SCORE_MAX = 0.80` in config and code default. Lines with QS ∈ [0.80, 0.85)
flip Noisy→Clear. Calibration `NOISY` fixtures only assert `!= Trash`, so they stay green;
`CLEAR` fixtures are only helped.

### Change 3 — all-caps headline word → scored, not Non-text
The single-token branch of `is_non_text` returns `Non-text` for any all-caps token with
`len ≥ 10` or containing `X` — exactly what traps `LITERATURA`. Add a guard: a single
**alphabetic** all-caps token whose `compute_vowel_ratio(tok) ≥ VOWEL_RATIO_LOW` (a real
headline word) returns `False` → `Process` → scored. Digit-bearing tokens and vowel-starved
all-caps (codes/garbage) stay `Non-text`. Reuses existing `compute_vowel_ratio` /
`VOWEL_RATIO_LOW (0.20)`. Multi-word all-caps headers already pass the pre-filter — unchanged.

### Changes 1 & 2 — "forgiven headline / abbreviation" floor
New pure helper `is_forgiven_headline(text, garbage_density)` classifies each token as:
- **STRUCTURAL** — pure digits / pure punctuation / roman numeral (`RE_ROMAN_NUMERAL`);
- **EXCEPTION** — normalized core (`lower`, drop `.`/`,`) in the new `SHORT_EXCEPTION_TOKENS`,
  or an `ACADEMIC_TITLES` title, or a `METADATA_MARKERS` marker, or in `SHORT_VALID_WORDS` /
  `SINGLE_CHAR_ALLOWED`;
- **CLEAN WORD** — passes the existing valid-word test (reuse `compute_valid_ratio`'s inner
  logic: `len(core) ≥ 3`, ≥70% alphabetic, no strange chars, not `_is_mid_uppercase`, vowel-bearing);
- **GARBAGE** — anything else.

Returns `True` iff `word_count ≤ HEADLINE_MAX_WORDS` **and** zero GARBAGE tokens **and** at least
one CLEAN WORD or EXCEPTION token **and** `garbage_density < CATEG_GARBAGE_DENSITY_HIGH`.
This is deliberately tight: any OCR-mangled token (`oAOrt`, `vyt1ačená`) disqualifies the line,
and high-perplexity garbage is still caught by the earlier hard-sweep routes.

Hooks:
- **`determine_category`** — compute `forgiven` once, *after* the hard-sweep / inverted /
  all-caps / garbage-density overrides (so genuine garbage is untouched). Then (a) skip
  `rule_short_garbage` (`:1012-1020`) when `forgiven`, and (b) in the QS-band Trash branch
  (`:1032-1038`) return `("Noisy","noisy_threshold")` instead of `Trash` when `forgiven`.
  **It only lifts Trash→Noisy — a forgiven line scoring ≥ 0.80 still routes to Clear (no
  demotion).** Emits `_fire("rule_forgiven_headline")` (new rule → coverage count 14→15).
- **`pre_filter_line`** — after the `METADATA_MARKERS` block (`:699-700`), before the Non-text
  gates, route `is_forgiven_headline` lines to `Process`. This lets bare `mm`/`cm` escape the
  `<4 chars → Non-text` gate (`:731`); pure-numeric/garbage lines are unaffected.

New config (`[TEXT_UTILS]`, loaded via `_get_csv_set`/`_get_int` near `LDL_UNITS`):
```
SHORT_EXCEPTION_TOKENS = mm,cm,m,g,kg,km,ha,l,ml,tb,neg,obr,str,č,čneg
HEADLINE_MAX_WORDS     = 8
```
And extend `METADATA_MARKERS` (`config_langID.txt:39`) with the missing comma/spacing variants
so single-token forms bypass Non-text: `č,neg`, `č,neg.`, `č.neg,`, `č,neg,`, `Tb,`, `tb,`,
`č, neg`. (Case-insensitive substring match already covers `tb.` via `Tb.`.) Do **not** put
2-char units in `METADATA_MARKERS` — substring match is too loose; units go through
`SHORT_EXCEPTION_TOKENS`.

Expected flips on the review lines: `2, Popis nálezu i - 3`, `4. Literatura 5`,
`7, Plánky 1 - 2`, `Plánek č. 1` → Trash→**Noisy**; `mm`/`cm`/`č.neg.` → Non-text/Trash→**Noisy**;
`LITERATURA` → Non-text→**scored**; clean prose and confirmed garbage unchanged.
