## Completed Phase 2 Work (Formerly "Remaining work")

### Increment 1 — B3: model-free importance backend (Morris → Sobol via SALib) (✅ DONE)

**Why it was worth doing despite conclusive-looking results:** the RF surrogate weakened after the
decouple — **oob_r² ≈ 0.684** (was ~0.81). Permutation importance from a moderately-fit surrogate
on 37 params / 19 docs deserved a **surrogate-free** confirmation. Morris/Sobol remove RF-MDI's
bias toward continuous params, and Sobol's **S1 vs ST** independently tests whether
`CATEG_GARBAGE_DENSITY_HIGH`'s 0.092 is genuine first-order sensitivity or interaction-driven —
the exact coupling question B2 was about.

Mirrored the existing backend contract in `tools/const_importance_sweep.py`
(`run_sklearn_backend(*, data, base_constants, params, output_dir, n_trials, seed, metric,
direction, eval_kwargs)`), reusing `evaluate_dataframe`, `objective_value`, `coerce_constants`,
`normalize_importances`, `save_json`, `save_importance_plot`, and `SEARCH_SPACE[name]["low"/"high"]`.

1. `requirements-sweep.txt`: added **`SALib`**.
2. Helper `_salib_problem(params)` created.
3. Helper `_constants_from_row(row, base_constants, params)` handled bounds and casting.
4. **Constraint handling**:
   - **Morris** screens **all 37**; *repairs* the ordering constraints (`CATEG_NOISY_SCORE_MAX`, `PERPLEXITY_THRESHOLD_MAX`).
   - **Sobol v1** restricts the variable set to the **constraint-free** constants.
5. `run_morris_backend(...)`: Implemented `morris.sample` and `morris.analyze(...)`.
6. `run_sobol_backend(...)`: Implemented modern `sobol.sample` (replaced deprecated `saltelli`) and enforced power-of-2 `n_trials`.
7. CLI: extended `--backend` choices with `morris`, `sobol`.
8. Added zero-variance statistical fallback guards.

**Files updated:** `tools/const_importance_sweep.py`, `requirements-sweep.txt`, `tests/test_sweep_backends.py`, `tools/run_optim_pipeline.sh`.
