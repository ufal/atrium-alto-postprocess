## 🎯 Goal & Core Constraint
Use a small surrogate model over the `[CLASSIFY]` / `[TEXT_UTILS]` config constants to find which actually move line categorization — **without** rerunning the GPU/FastText/XML pipeline.
* **The Approach:** The input per-line CSVs are treated as immutable ground truth. The offline tools preserve the current categories, train a surrogate to predict them, and vary the constants across trials.
* **The Self-Reference Trap:** Because the target `categ` labels are generated by the current config, the **baseline macro-F1 = 1.0 and flip-rate = 0 by construction**. The sweep correctly maps the *sensitivity* of the pipeline ("which constants **can** change the current categorization"), but it cannot prove what is objectively **better** for OCR quality. That requires independent gold labels.

## 🛠️ Tooling Built & Refactored
* **`recategorize_from_csv.py`**: Faithful offline re-scorer. Replays scoring/categorization from existing CSV features. This keystone trick severs the cheap categorization math from the expensive GPU/LLM extraction.
* **`const_importance_sweep.py`**: Samples config constants, fits a sklearn RandomForest surrogate (with an optional Optuna/fANOVA backend), and emits parameter-importance rankings via MDI/permutation.
* **`run_ablation_study.py`, `greedy_backward_elimination.py`, `ab_constant_eval.py`**: Tools to test which rules/constants can be removed or changed.
* **Refactor `b4bd545` (Rule/Config Surface)**:
    * **Removed:** Dead code including `rule_short_fragment_noisy`, `CLEAR_BAND_WC_MIN`, and legacy `CLEAN_PROSE_*` constraints.
    * **Parameterized:** 8 page-context smoothing thresholds (e.g., `SURROUNDED_TRASH_QS_MARGIN`, `PAGE_CLEAN_CLEAR_MIN`).
    * The sweep search space is now **36 parameters** (up from 33), and tests for the low-PPL fast-track were added.

## 🏆 Headline Findings & Conclusions

**1. The "Big Four" Dominate**
A handful of readability and perplexity gates control ~84% of the categorization variance. The system is genuinely over-parameterized; the 9 `QS_WEIGHT_*` parameters are non-identifiable from the labels and sit in the noise floor (~0.001–0.008) alongside dozens of other constants.
* `MOSTLY_READABLE_VALID_MIN` (~0.46 - 0.528): Dominant Clear/Noisy readability gate.
* `LOWPPL_CLEAR_MAX` (~0.21 - 0.238): Low-perplexity Clear rescue.
* `CATEG_GARBAGE_DENSITY_HIGH` (~0.128): Garbage-density cutoff.
* `LOWPPL_CZECH_CLEAR_MAX` (~0.051 - 0.06): Czech-specific low-PPL rescue.

**2. The "Delete 15 Rules" Metric Artifact Fixed**
An initial run concluded all 15 heuristics were deletable. This was a metric artifact caused by a cost proxy (`min(clear_drop, trash_rise)`) that missed one-directional leakage (e.g., Trash leaking into Noisy). Switching to a true per-line confusion matrix + macro-F1 frontier revealed:
* **3 rules are strictly redundant** on this corpus (zero flips, zero F1 cost: `rule_garbage_density`, `rule_inverted`, `rule_allcaps`).
* **8 rules are prunable** within a 0.2% F1 tolerance.
* **6 rules are load-bearing** (`penalty_ledger_fragmentation`, `penalty_wqx_rot`, `rule_extreme_ppl`, `rule_lowppl_clear`, `rule_mostly_readable_noisy`, `rule_trailing_fill_rescue`).
* *Conclusion:* Do not delete the 9 dead rules yet. They are likely "corpus-redundant," not dead code. These cheap structural guards should stay until multi-collection coverage evidence exists.

**3. The `CATEG_GARBAGE_DENSITY_HIGH` Confound**
The highest-importance actionable move (0.35 → 0.55) is conflated because the constant is reused in three places (the hard gate, the QS garbage normalization, and the short-line penalty). The A/B test claiming 0.55 is "entirely safe" is overstated; it only shows deviation from self-generated labels. It remains a candidate needing manual review.

**4. The Baseline is Near-Optimal (Against Itself)**
The system is already within ~2% of the surrogate's optimum *on this corpus* with KL ≈ 0. Blindly retuning or copying `best_config.json` has little upside, as low-importance parameters in the optimized config are just statistical noise.

**5. Cross-Backend Consensus & Rotation Guard**
With the addition of the Sobol backend, the cross-backend consensus confirms the "Big Four" parameters dominate
categorization and that the garbage gate's leverage is a first-order effect, not an interaction artifact. A dedicated
Czech-prose rotation regression suite (`tests/test_rotation_regression.py`) now acts as a strict gating test, ensuring
that no tuning of `#5` constants across their swept bounds will ever demote valid Czech text to `Trash`.

## 🏁 Final Bottom Line
Issue #5 successfully produced a trustworthy sensitivity map and reusable laptop-runnable infrastructure, and it cleaned up provably dead code. However, recalibration is impossible while labels are self-referential.
**Resolution:** Ship the tooling and the dead-code refactor (`b4bd545`), freeze the defaults, keep the current rules, and treat the garbage-gate move as a manually-reviewed candidate. The phase of "tuning harder" is over; the focus must shift to "breaking the self-reference."

## 🚀 Expansion Plan & Future Work (High Leverage)
1.  **Break the self-reference with a gold set:** Add `tools/build_label_queue.py` as an active-learning extractor targeting high-information lines (flips, decision margins, rule boundaries). Update `evaluate_dataframe` with an optional `gold_column` to score *only* against these human-annotated labels.
2.  **Decouple `CATEG_GARBAGE_DENSITY_HIGH`:** Split the overloaded constant into `CATEG_GARBAGE_DENSITY_HIGH` (for the hard gate) and `QS_GARBAGE_NORM_MAX` (for the QS scaling). Default both to 0.35 to maintain bit-for-bit parity, then A/B them independently and add the new constant to the sweep space.
3.  **Model-free sensitivity backend (Sobol/Morris):** Add `--backend morris|sobol` (via SALib) to `const_importance_sweep.py` to remove RF-MDI bias toward continuous/high-cardinality parameters and expose coupling.
4.  **Full Corpus & Consensus:** Force reporting per-document (`baseline_per_document.json`). Treat importance rankings as final only when MDI, permutation, and Sobol broadly agree across the full cluster corpus.
5.  **Coverage-gated rule retirement:** Retire rules only if they are both corpus-redundant *and* coverage-empty across multiple distinct collections.
6.  **Replace threshold stack with an interpretable model (Longer term):** Once a gold set exists, swap the hand-tuned cascade for a monotonic, interpretable classifier (e.g., monotonic GBM) that learns boundaries from human truth and can natively encode cost-sensitive logic (Clear→Trash loss is worse than Trash→Noisy).
