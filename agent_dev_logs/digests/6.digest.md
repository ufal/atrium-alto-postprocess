# 📓 Digest — [\#6](https://github.com/ufal/atrium-alto-postprocess/issues/6) `open`: Add starting points to the pipeline run script

**Labels**: `enhancement`, `development`

## 🎯 Goal & Core Constraint
Enable stage skipping and custom starting points in the `run_pipeline.py` orchestrator. The primary objective is to allow the pipeline to skip expensive operations (such as GPU-bound text extraction or page splitting) when the intermediate files already exist on disk.

## 🏆 Headline Findings & Conclusions

**1. Comprehensive Skip Flags Implemented**
The orchestrator now supports targeted exclusion of any stage via CLI flags (`--skip-split`, `--skip-stats`, `--skip-extract`, `--skip-classify`, `--skip-aggregate`) and configuration parameters.

**2. The `--start-from` Convenience Wrapper**
A `--start-from` argument was introduced to dynamically skip all stages prior to the requested starting point. This enables seamless resumption of interrupted workflows.

**3. Paradata and Environment Propagation**
The pipeline correctly handles logging for skipped stages. Skipped steps emit no paradata, and the final merged run summary explicitly lists `skipped_stages`, ensuring the effective output license reflects only the stages that actually executed. Furthermore, directory references like `LANGID_TEXT_DIR` are successfully propagated downstream even when the extraction stage is bypassed, preventing the classifier from defaulting to the wrong path.

## 🏁 Final Bottom Line
The feature is fully implemented and auto-tested in the codebase. The issue remains open solely for practical verification on a full collection run to ensure downstream stages consume the pre-existing files without error.
