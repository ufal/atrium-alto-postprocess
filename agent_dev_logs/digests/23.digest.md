## Issue #23 — Fine-tuning an NLP model for quality-score assignment · Digest (2026-07-22)

**Issue:** [ufal/atrium-alto-postprocess#23](https://github.com/ufal/atrium-alto-postprocess/issues/23) · open · labels `enhancement`, `development`, `a big one` · milestone **Q4 [WP3: Conclusion of Semantic Harmonization]** · author K4TEL · **no upstream comments yet.**

## The ask
Distil the hand-crafted OCR line-quality engine (FastText + Qwen2.5-0.5B perplexity + a 9-signal weighted score) into **one small regression model** (score 0.0–1.0, smaller than Qwen-0.5B). Manufacture training-label diversity by corrupting `Clear` lines → `Noisy`/`Trash`, auto-correcting `Noisy` lines → semi-`Clear` (korektor for Czech; optionally GLM), using `Trash`/`Non-text` as-is, ignoring `Empty` — and **check the algo score delta after correction**.

## Governing constraints (from dev logs)
- **Self-reference trap (#5):** labels come from the algorithm itself → a model can only *distil* it; "better" is provable only on the **expert gold subsets** (2,171-line eval corpus, 1,348-line review sample). Gold-set training is deferred.
- **Category ≠ f(score) (#3):** structural overrides flip categories independently of the score → model needs a **category aux head**; rules stay in production.

## The end-to-end pipeline (all reusing ONE production scoring engine)
`corrupt.py` (Clear → Noisy/Trash variants) → `build_dataset.py` (select → generate → **relabel** → dedup → split-by-document, gold docs held out → balance) → `correct.py` (korektor / GLM on Noisy lines) + `report_correction_delta.py` (the issue's Δ check) → `train_baseline_gbm.py` (GBM floor) + `train.py` (DistilBERT fine-tune) → `evaluate.py` (vs-algorithm metrics + the gold gate). Relabelling is done by `score_texts.build_line_record`, a faithful mirror of production; `common.py` holds the shared config/metric/banding helpers (band thresholds read from `text_util_langID`, never hardcoded).

## Key design decisions locked
- **D1** regress the **raw pre-clamp** score (smooth) + 3-way category aux head; store the clamped score too.
- **D2** relabel every row with a fresh FastText+Qwen pass — never the stored clamped score.
- **D6** primary model **`distilbert-base-multilingual-cased`** (134M); CANINE-s fallback; ≤5M char-CNN student as the stretch prize (CPU-side, kills the GPU-worker bottleneck).
- **D7** GBM baseline trained twice (± perplexity) — the −perplexity number is the floor the encoder must beat to justify dropping Qwen.
- **D8** loss = Huber(0.1) + 0.3·CE. **D9** korektor via LINDAT REST (`czech-spellchecker-130202`) / local binary / pluggable GLM; correction is offline data-gen only, so the CC BY-NC-SA model licence affects provenance notes, not runtime.
- **One-engine invariant:** every tool reuses production `text_util_langID` functions — no second scorer.

## Verified behaviour worth noting
- Corruption ops each provably move their target detector; the dataset builder has no doc-leakage across splits, holds gold docs out, dedups, and is reproducible; offline monotonicity ≈ 0.81 on the sample corpus.
- The correction delta report cleanly exposes that **diacritic-only fixes show ~0 Δ under the offline scorer** (fixed perplexity) — so the **real gate requires `--scorer model`** (real Qwen perplexity penalizes the garbled source). Documented, not a bug.
- `evaluate.py` runs end-to-end on a predictions CSV: regression, banded + category-head agreement, calibration, predicted-score monotonicity, per-language stratification, and the **gold gate** (a smoke run scored model macro-F1 1.0 vs algorithm 0.556 → PASS).
