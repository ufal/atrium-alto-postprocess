## 🎯 Goal
Calibrate the line-categorization logic (`Clear` / `Noisy` / `Trash` / `Non-text` / `Empty`) over the parameters extracted from each text string. Baseline at open: **v0.13.0**. A long iterative loop with DanaKriv (and the ARUP/ARUB teams) reviewing sample CSV outputs.

## 🔧 Fixes that landed (by commit)
**`fddbafd` — structural detectors & thresholds**
- Killed the aggressive OCR word-split repair regex in `pre_filter_line` that merged single-letter Czech prepositions (`obilost k službě` → `obilostkslužbě`); the 7 broken token patterns now return 0 hits.
- Repeat detection: lowered the abnormal-distribution threshold 0.40 → 0.30, added a consecutive-doubles check (`tt`, `CC`) to catch garble (`ARCaACONCCN`, `Neríxto^ťtt`).
- Gated `rot_penalty` in `compute_quality_score` on `weird_ratio > 0` so clean short Czech labels (`Osada:`, `Okres:`) stop being dumped into Trash on their naturally high short-text perplexity.
- Raised `CATEG_NOISY_SCORE_MAX` → **0.90** (hold internal-noise lines in Noisy) and `CATEG_TRASH_SCORE_MAX` 0.45 → **0.50**.
- New output cols `fused_words`, `vowel_ratio`, `rot_ratio`; `langID_aggregate_STAT.py` adds `num_lines` and reorders to `Clear, Noisy, Trash, Non-text, Empty`. (Sample outputs in `140920e`.)

**`646fc5e` — scoring model**
- Language trust encoded directly in the word score; `quality_score` becomes the **sole** categorization factor; model swap + language remapping.

**`4dfc084` — decoupling & overrides**
- Decoupled `rot_ratio` from weirdness in the penalty calc → inverted/upside-down Czech scans correctly drop `Noisy → Trash` (`CTX192100040` L18/L21).
- Passed `garbage_density` to the hard override → punctuated gibberish (`CTX192601143` L50/L240) forced back to `Trash`.
- Removed the `symbol` penalty, redistributed weight to `valid_word_ratio` / weirdness → clean prose promoted `Noisy → Clear` (`různé prameny.`, L226). Fixed the `x`-bug that penalized valid Czech words.
- Later pass (2026-06-15): per-rule diagnostic boolean columns + multi-char symbol under-counting fix.

## ⏳ Open cases (flagged "to be confirmed by a Czech speaker")
- **Vowelless illusion** — `WVL A`, `ClAŕ,` survive as `Clear` (qs ≈ 0.85) because `score_word` trusts alpha blocks. Proposed: phonetic-anchor penalty for 3+ char tokens with zero vowels (non-abbreviation) + heavier mid-uppercase penalty.
- **Ledger / table loophole** — `nonč, mI 47 žn dn A' 1074 484 "Nik 1726.` scores ≈ 0.85 because individual number/word tokens look fine. Proposed: line-level fragmentation penalty (ratio of short/digit tokens → spike weirdness).
- **Symbol cluster** — `At . O/wvi` sits at Noisy; the vowelless penalty should combine with the existing `w` penalty to push it to Trash.
- **Collateral (working as intended):** `Maždý…` (typo drops `valid_word_ratio` → Clear→Noisy) and `P.T. panu` (2-word, high punct → Noisy→Trash) are correct QS behavior, not regressions.
