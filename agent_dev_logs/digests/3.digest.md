- New output cols `fused_words`, `vowel_ratio`, `rot_ratio`; `langID_aggregate_STAT.py` adds `num_lines` and reorders to `Clear, Noisy, Trash, Non-text, Empty`. (Sample outputs in `140920e`.)

**`4dfc084` — decoupling & overrides**

## 📞 Calibration pass — 2026-07-02 (DanaKriv meeting)
DanaKriv flagged five new patterns from a live review session, verified against `CTX199603106` (a Charles-Bridge archaeological report):
1. **Short domain abbreviations** (`mm`, `cm`, `Tb.`, `tb.`, `č.neg.`, comma/case variants) land in `Trash`/`Non-text` — should be `Noisy`. Units glued to digits (`14mm`) already correct.
2. **Short numbered headlines/captions** (`2, Popis nálezu i - 3`, `4. Literatura 5`, `7, Plánky 1 - 2`, `Plánek č. 1`) land in `Trash` — should be `Noisy`.
3. **All-caps single-word headlines** (`LITERATURA`) marked `Non-text` — should be scored normally.
4. **Noisy↔Clear boundary** `0.85 → 0.80`.
5. **Extra**: remapped lang score should be **always** 0.75/0.5 — *"the original lang score should not matter."*

## ⏳ Open cases (flagged "to be confirmed by a Czech speaker")
- **Vowelless illusion** — `WVL A`, `ClAŕ,` survive as `Clear` (qs ≈ 0.85) because `score_word` trusts alpha blocks. Proposed: phonetic-anchor penalty for 3+ char tokens with zero vowels (non-abbreviation) + heavier mid-uppercase penalty.
- **Ledger / table loophole** — `nonč, mI 47 žn dn A' 1074 484 "Nik 1726.` scores ≈ 0.85 because individual number/word tokens look fine. Proposed: line-level fragmentation penalty (ratio of short/digit tokens → spike weirdness).
- **Symbol cluster** — `At . O/wvi` sits at Noisy; the vowelless penalty should combine with the existing `w` penalty to push it to Trash.
- **Collateral (working as intended):** `Maždý…` (typo drops `valid_word_ratio` → Clear→Noisy) and `P.T. panu` (2-word, high punct → Noisy→Trash) are correct QS behavior, not regressions.
